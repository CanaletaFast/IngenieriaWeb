Diseño Responsivo (CSS)

En esta sección, aprenderemos los fundamentos del diseño responsivo utilizando CSS, permitiendo que nuestras páginas web se adapten a diferentes tamaños de pantalla y dispositivos.

Ejemplo 1: Media Queries Básicas

@media (max-width: 600px) {
    body {
        background-color: lightblue;
    }
}

Resultado:

Este ejemplo cambia el color de fondo a azul claro cuando el ancho de la pantalla es de 600px o menos.

¡Hola! Vamos a entender cómo usar media queries básicas en CSS.

¿Qué es una media query?

Las media queries permiten aplicar estilos CSS específicos según las características del dispositivo, como el ancho de la pantalla.

Entendiendo el código:

@media (max-width: 600px) {
    body {
        background-color: lightblue;
    }
}

Elementos clave:

  • @media (max-width: 600px) { ... }: Aplica los estilos dentro del bloque solo si el ancho de la pantalla es de 600px o menos.
  • background-color: lightblue;: Cambia el color de fondo del cuerpo a azul claro.

Resultado visual:

Al reducir el tamaño de la ventana del navegador a 600px o menos, el fondo de la página cambia a azul claro.

¡Hola! Vamos a crear layouts responsivos usando Flexbox en CSS.

¿Qué es Flexbox?

Flexbox es un modelo de diseño de CSS que facilita la creación de layouts flexibles y responsivos, alineando elementos de manera eficiente.

Entendiendo el código:

.container {
    display: flex;
    flex-wrap: wrap;
}

.item {
    flex: 1 1 200px;
    margin: 10px;
    background-color: #f2f2f2;
    padding: 20px;
    box-sizing: border-box;
}

Elementos clave:

  • display: flex;: Activa el modo Flexbox en el contenedor.
  • flex-wrap: wrap;: Permite que los elementos flexibles se envuelvan a la siguiente línea si no caben en una sola fila.
  • flex: 1 1 200px;: Define que los elementos pueden crecer y encogerse, con un tamaño base de 200px.
  • box-sizing: border-box;: Incluye el padding y el border en el tamaño total del elemento.

Resultado visual:

Los elementos dentro del contenedor se alinean en filas flexibles, adaptándose al tamaño de la pantalla. Al reducir el ancho, los elementos se envuelven a nuevas filas.

¡Hola! Vamos a hacer que las imágenes sean responsivas en CSS.

¿Qué significa una imagen responsiva?

Una imagen responsiva se adapta automáticamente al tamaño de su contenedor, evitando desbordamientos y manteniendo la proporción.

Entendiendo el código:

img.responsive {
    max-width: 100%;
    height: auto;
    display: block;
}

Elementos clave:

  • max-width: 100%;: Limita el ancho máximo de la imagen al 100% de su contenedor.
  • height: auto;: Ajusta la altura automáticamente para mantener la proporción.
  • display: block;: Elimina espacios en línea alrededor de la imagen.

Resultado visual:

La imagen se ajusta automáticamente al ancho del contenedor sin perder su proporción, adaptándose a diferentes tamaños de pantalla.

fetch('https://jsonplaceholder.typicode.com/invalid-endpoint')
    .then(response => {
        if (!response.ok) {
            throw new Error('Network response was not ok');

        return response.json();

    .then(data => {

¡Hola! Vamos a manejar errores al usar Fetch en JavaScript.

¿Qué es Fetch?

Fetch es una API de JavaScript para realizar solicitudes HTTP de manera asincrónica.

Entendiendo el código:

fetch('https://jsonplaceholder.typicode.com/invalid-endpoint')
    .then(response => {
        if (!response.ok) {
            throw new Error('Network response was not ok');
        }
        return response.json();
    })
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error('Hubo un problema con la solicitud Fetch:', error);

Elementos clave:

  • fetch('URL'): Realiza una solicitud a la URL especificada.
  • response.ok: Verifica si la respuesta fue exitosa.
  • throw new Error('...');: Lanza un error si la respuesta no es exitosa.
  • .catch(error => { ... });: Maneja cualquier error que ocurra durante la solicitud.

Resultado visual:

Al ejecutar este código, se generará un error debido a una URL inválida. El error se manejará en el bloque catch y se mostrará en la consola del navegador.

Ejemplo 9: Autocompletar con jQuery UI y Ajax

$(function() {
    $("#buscar").autocomplete({
        source: function(request, response) {
            $.ajax({
                url: "https://jsonplaceholder.typicode.com/users",
                method: "GET",
                dataType: "json",
                success: function(data) {
                    const resultados = data.filter(user => user.name.toLowerCase().includes(request.term.toLowerCase()));
                    response(resultados.map(user => user.name));
                },
                error: function() {
                    response([]);
                }
            });
        },
        minLength: 2,
    });
});
